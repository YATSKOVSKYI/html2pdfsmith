# Html2PdfSmith

Browserless HTML-to-PDF rendering engine for TypeScript and Bun.

This is intentionally not a full browser or a full CSS engine. It targets a practical document subset:

- A4 portrait/landscape pages
- tables with many columns
- table border, padding, border-collapse, colspan, and rowspan handling
- repeated headers
- page header/footer and page numbers
- section rows and highlighted cells
- header logo/text
- text blocks, contact text, QR images
- PNG/JPEG logos
- text or image watermarks
- optional `qpdf` owner-password protection

The library is designed for Bun and avoids Chromium/Playwright memory overhead.

## Usage

```ts
import { renderHtmlToPdf } from "html2pdfsmith";

const pdf = await renderHtmlToPdf({
  html,
  repeatHeaders: true,
  pageHeader: { text: "Quarterly Report", align: "right" },
  pageFooter: { text: "Generated by Html2PdfSmith", align: "left" },
  pageNumbers: { format: "Page {page}", align: "right" },
  watermarkUrl: userLogoUrl,
  userLogoUrl,
  font: {
    regularPath: "/usr/share/fonts/truetype/noto/NotoSansCJK-Regular.ttc",
    boldPath: "/usr/share/fonts/truetype/noto/NotoSansCJK-Bold.ttc",
  },
});

await Bun.write("table.pdf", pdf);
```

## Current Scope

Supported HTML is document-oriented with strong table support: `header`, `h1`-`h6`, `p`, `ul`, `ol`, `li`, `hr`, `img`, `table`, `thead`, `tbody`, `tr`, `th`, `td`, `br`, and text nodes. CSS parsing is deliberately shallow; common printable styles and inline styles are recognized.

Unsupported CSS is ignored. This is a renderer for predictable document templates, not arbitrary web pages.

## Memory Policy

The default renderer is streaming and does not auto-load large system fonts. This keeps Latin-only jobs light. For Chinese, Japanese, Korean, Cyrillic, Arabic, and other scripts, pass explicit fonts. For production memory targets under 100 MB incremental RSS, prefer small subset fonts instead of full CJK variable fonts.
